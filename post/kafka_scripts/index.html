# 创建 Topic
bin/kafka-topics.sh --zookeeper zk_host:port \
--create \
--topic my_topic_name \
--partitions 20 \
--replication-factor 3 \
--config x=y
# replication-factor 不能大于当前可用的 broker数

# 修改 Topic
bin/kafka-topics.sh --zookeeper zk_host:port \
--alter \
--topic my_topic_name \
--partitions 40
# partition 数量只能增加不能减少

kafka-configs.sh

修改配置

1
2
3
4
5

bin/kafka-configs.sh --zookeeper localhost:2181 \
--entity-type topics \
--entity-name my-topic \
--alter \
--add-config max.message.bytes=128000
  • entity-type 指定修改配置的层级
  • entity-name 指定该层级下的某个实体，这里层级是 Tpoic 则对应的就是 Topic Name
  • alter 指定操作类型
  • add-config 操作类型为 增加 (max.message.bytes=128000 )

如果配置存在需要修改也是用 --add-config 覆写

查看配置

--entity-type topics \
--entity-name my-topic \
--describe

--entity-type topics \
--entity-name my-topic \
--alter \

kafka-mirror-maker.sh

集群镜像工具，用来迁移数据，原理就是在源集群这边放生产者，在目的集群放消费者，因此目的集群的 partition 数、 offset 可以和源集群的不一样

1
2
3
4

bin/kafka-mirror-maker.sh \
--consumer.config consumer.properties \
--producer.config producer.properties \
--whitelist my-topic
  • whitelist: 需要迁移的 Topic，支持正则表达式， 也可以使用 blacklist（new consumer 不能使用，默认是 old consumer， 通过 --new.consumer 可以使用新消费者客户端）

kafka-reassign-partitions.sh

1
2
3
4

重新分配分区的工具，具有三个模式：
- --generate: 给定 Topic 列表(`--topics-to-move-json-file`)和 Broker 列表(`--broker-list`)， 生产 partition 分配方案
- --execute: 根据分配方案执行分区的迁移，方案可以是 `--generate` 生成的也可以自定义， 通过 `--reassignment-json-file` 指定
- --verify：查看分区迁移的执行情况， 可以是 successfully completed, failed 或者 in progress

kafka-producer-perf-test.sh

生产端性能测试

  • –topic： topic 名称
  • –num-records： 消息数量
  • –throughput： 吞吐量，每秒最多发送的消息数量
  • –record-size： 消息大小
  • –payload-file： 消息来源文件
  • –payload-delimiter： 消息来源文件中分隔消息的符号（默认为换行符）
  • –print-metrics： 测试结束后显示统计数据，默认 false
  • –producer-props： 生产者相关配置 PROP-NAME=PROP-VALUE， 例如 bootstrap.servers 等等
  • –producer.config： 存有生产者相关配置的文件

注意：

  1. –record-size 和 –payload-file 必须有一个但不能都有
  2. –producer-props 的优先级高于 –producer.config

kafka-topics.sh

  • list topics bin/kafka-topics.sh --zookeeper localhost:2181 --list

  • show topic info: bin/kafka-topics.sh --zookeeper 172.16.16.8:2181 --describe --topic kvision-log

  • create topic: bin/kafka-topics.sh --zookeeper 172.16.16.8:2181 --create --replication-factor 2 --partitions 4 --topic test_created_topic

  • increase topic: (消费者数量⼤大于⽬目前tpoic的partition个数，则需要增加此topic的partition个数) bin/kafka-topics.sh --alter --zookeeper localhost:2181 --partitions 8 --topic test_created_topic

  • modify topic retention time: bin/kafka-topics.sh --zookeeper 10.19.24.13:2181 --alter --topic kvision-log --config retention.ms=3600000

kafka-reassign-partitions.sh

重新分配分区的工具，具有三个模式：

  1. generate: 给定 Topic 列表(–topics-to-move-json-file)和 Broker 列表(–broker-list)， 生产 partition 分配方案
  2. execute: 根据分配方案执行分区的迁移，方案可以是 –generate 生成的也可以自定义， 通过 –reassignment-json-file 指定
  3. verify：查看分区迁移的执行情况， 可以是 successfully completed, failed 或者 in progress

例如测试环境下对test_created_topic进行重分片，具体操作：

  1. 使用Kafka-topics.sh查看当前topic的分片状态，可以看到test_created_topic只分配在1001和1002两台broker上,现在想将此topic分配到1001，1002和1003上，首先手写一个json文件，列出想要重新分配的topic
  • topiclist.json
1
2
3
4

{
"topics": [{"topic": "test_created_topic"}],
"version": 1
}
  1. 执行指令
1
2
3

./bin/kafka-reassign-partitions.sh --zookeeper localhost:2181 \
--topics-to-move-json-file topiclist.json \
--broker-list "1001,1002,1003" --generate
  • 指令执行完毕，会打印出如下信息：

Current partition replica assignment
 
{"version":1,"partitions":[{"topic":"test_created_topic","partition":0,"replicas":[1001,1002]},{"topic":"test_created_topic","partition":1,"replicas":[1002,1001]},{"topic":"test_created_topic","partition":3,"replicas":[1002,1001]},{"topic":"test_created_topic","partition":2,"replicas":[1001,1002]}]}
 
Proposed partition reassignment configuration
 
{"version":1,"partitions":[{"topic":"test_created_topic","partition":0,"replicas":[1003,1001]},{"topic":"test_created_topic","partition":1,"replicas":[1001,1002]},{"topic":"test_created_topic","partition":3,"replicas":[1003,1002]},{"topic":"test_created_topic","partition":2,"replicas":[1002,1003]}]}

Current partition replica assignment表示当前的消息存储状况。Proposed partition reassignment configuration表示迁移后的消息存储状况。 将迁移后的Proposed partition reassignment configuration这一部分的json存入一个文件 例如re.json，供–execute命令使用。

  1. 执行–execute bin/kafka-reassign-partitions.sh --zookeeper localhost:2181 --reassignment-json-file re.json --execute

  2. 可以查看观察进度 /bin/kafka-reassign-partitions.sh --zookeeper localhost:2181 --reassignment-json-file re.json --verify

  3. 确认完成： bin/kafka-topics.sh --zookeeper 172.16.16.8:2181 --describe --topic kvision-log
