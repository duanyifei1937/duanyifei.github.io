curl -X POST <M3_COORDINATOR_IP_ADDRESS>:<CONFIGURED_PORT(default 7201)>/api/v1/namespace -d '{
  "name": "default_unaggregated",
  "options": {
  	# 启动dbnode时，同时启动namespace;
    "bootstrapEnabled": true,
    # 周期性flush blocks to disk;
    "flushEnabled": true,
    # wirte commitlog;
    "writesToCommitLog": true,
    "cleanupEnabled": true,
    # write snapshot file;
    "snapshotEnabled": true,
    # 对比并修复数据；
    "repairEnabled": false,
    "retentionOptions": {
    	# namespace保留数据时长；
      "retentionPeriod": "2d",
      # m3db性能关键参数：更大的值消耗更多内存，但高压缩比；
      # 推荐：根据保留时间推荐block size;
      "blockSize": "2h",
      # 可接受的时间序列与当前系统时间的相对范围；
      # 更大的范围造成性能压力；乱序写，并需要在内存中保留更长时间；
      "bufferFuture": "10m",
      "bufferPast": "10m",
      "blockDataExpiry": true,
      "blockDataExpiryAfterNotAccessedPeriod": "5m"
    },
    "indexOptions": {
      "enabled": true,
      "blockSize": "2h"
    }
  }
}

m3aggregator deploy

aggregator部署方式docs隐藏的比较深：aggregator

m3coordinators –> m3aggregators –> m3coordinators –> m3db

m3coordinators 将未聚合merics发送给m3aggregators进行数据聚合，聚合后数据回退给m3coordinators， 再写入m3db;

需要m3coordinators增加配置接受回馈信息；

暂时不使用，直接使用m3coordinators 聚合功能；
