Cdn 跨域

2021-01-19 584 words 2 mins read

Contents

  • 原因
  • 解决

CDN跨域问题，之前每次刷新解决。。。

原因

  • oss 存放 abc.js 文件；
  • 第一次携带’Origin: baidu.com’ 请求cdn节点:
    • cdn节点miss，回oss拉取资源;
    • oss返回head信息：‘access-control-allow-origin: baidu.com';
    • cdn带’access-control-allow-origin: baidu.com’ 缓存资源；
  • 第二次请求携带’Origin: sina.com’ 请求cdn节点:
    • 却受到’access-control-allow-origin: baidu.com’ 信息，造成跨域不通过；

解决

  1. oss 直接返回’access-control-allow-origin: *‘；

    1. cdn为前端资源，安全级别不高；
    2. 减少增加域段配置；
    3. 需要将oss跨域来源修改为*，刷掉cdn上所有资源，重新缓存为 access-control-allow-origin: *， 因此不能完全依赖oss配置；

  3. cdn增加配置’access-control-allow-origin: *‘；

    1. 保证访问cdn的全部请求都可以拿到access-control-allow-origin: *；

  4. access-control-allow-origin: * 出现了以下问题：

    2. credentials: true 开启预检模式，不允许allow-origin: *; “withCredentials :true”
      1. credentials: true 不允许allow-origin: *;
      2. credentials: false 不允许携带cookie;
      3. js 做正则判断，
        1. 针对’js/json/css‘等， credentials：false请求，
        2. 后端api true 请求；
