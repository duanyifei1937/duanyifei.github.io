原因

使用了错误的mapping, 导致无法写入;
由于使用动态mapping, mapping的格式完全依赖首次写入的数据格式，并且两种格式无法共存；
dynamic mapping可动态添加new field, 不能修改存在field的type;
Dynamic field mappingedit
By default, when a previously unseen field is found in a document, Elasticsearch will add the new field to the type mapping.

"warning": 1051

"warning": {warningCode=1051}
