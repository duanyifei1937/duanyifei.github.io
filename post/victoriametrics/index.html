对比m3db/thanos, 确认VictoriaMetrics 以下问题：

数据去重

当multiple prometheus 同时监控相同target;

由于时钟间隔造成的重复数据；

  • 查数据时去重：
    • thanos/m3db 相同指标数据增加external_labels不同label, 查询时忽略查询(数据缺失可部分响应；)
  • 写数据时去重：
    • victoria 写入时忽略相同metrics(保证external_labels相同)；特定时间间隔内重复写入丢弃；

数据量存储大小问题

  • m3db降样本数据存储成本过高，1min + 5min 几乎等价不压缩数据，并且3RF, 投票写入；
  • Victoria 无上层降样本数据，存储成本 == prometheus存储；

Victoria 增、删节点流程

暂时未提供rebalance功能；vmctl

多种保留时间

路由到不同retention的节点；

- 增加写入性能：
# memory usage = number_of_shards * (capacity + max_samples_per_send)
# 30 * 100KB = 3M
# capacity: per shard have number of samples;
# 减少发送次数，增加每次数量；
  remote_write:
  - url: http://<victoriametrics-addr>:8428/api/v1/write
    queue_config:
      max_samples_per_send: 10000
      capacity: 20000
      max_shards: 30
