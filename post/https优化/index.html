HTTPS优化

谈到https优化，先得知道一次https中间经过了哪些动作？哪些地方可以优化？

HTTPS访问过程

可优化点

  1. server - 产生用于密钥交换的临时公私钥对（ECDHE）；
  2. client - 验证证书时访问 CA 获取 CRL 或者 OCSP；
  3. 非对称加密解密处理“Pre-Master”。

硬件优化：
  - CPU(AES支持)；
  - SSL加速卡；
  - SSL加速服务器(专门的服务器用来加解密)；

- 软件优化：
  - 软件升级
    - openssl
    - kernel
    - nginx-version

  - 协议优化：
    - TLS1.3 -- 1RTT
    - TLS1.2 -- (握手时使用的密钥交换协议应当尽量选用椭圆曲线的 ECDHE 算法)

ssl_ciphers TLS13-AES-256-GCM-SHA384:TLS13-CHACHA20-POLY1305-SHA256:EECDH+CHACHA20；(算法选择)
ssl_ecdh_curve X25519:P-256;(椭圆曲线)


- 证书优化：
  - CRL：废弃证书序号列表，文件过大；
  - OCSP： client每次访问ca,获取ocsp response;
  - OCSP stapling: 服务器预先访问ca或者ocsp响应，随证书一起发送client；

- 会话复用：
  master secret 不更换(session保存在server)

- session ticket(保存在client)

- 预共享秘钥

开启OCSP staping
